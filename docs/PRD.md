# 맛집 룰렛 (Matjip Roulette) — PRD

## 1. 제품 개요

**맛집 룰렛**은 "오늘 뭐 먹지?" 라는 결정을 짧고 재미있는 경험으로 바꿔주는 웹 서비스입니다.

사용자는 위치(현재 위치 또는 직접 선택한 지역)만 정해주면, 맛집 룰렛이 후보 식당을 자동으로 모아 8칸짜리 원형 룰렛에 올려둡니다. 사용자가 룰렛을 돌리면 한 곳이 당첨되어 사진·주소·전화번호와 함께 지도 링크가 공개됩니다. 마음에 들지 않으면 즉시 다시 돌릴 수 있습니다.

검색과 추천 알고리즘에 지친 사용자에게 **"고민하는 시간보다 결정의 재미"** 를 돌려주는 것이 목표입니다.

---

## 2. 문제 정의

직장인·여행자·모임 참석자 모두 동일한 문제를 매일 마주합니다.

- **결정 피로 (Decision Fatigue)**: 점심·저녁마다 "어디서 먹을까" 를 반복적으로 결정해야 합니다. 선택지가 많아질수록 만족도는 오히려 떨어집니다.
- **필터 버블**: 평소 가던 식당, 평소 검색하던 키워드 안에 갇혀 새로운 곳을 시도하지 못합니다.
- **모임에서의 결정자 부재**: 단체 자리에서 "아무거나" 가 반복되며 결정이 미뤄집니다.
- **낯선 지역에서의 막막함**: 출장, 여행, 새 동네 방문 시 어디부터 봐야 할지 막막합니다.

기존 검색·리뷰 서비스는 "정보를 더 많이 보여주는" 방향으로 문제를 해결하려고 합니다. 맛집 룰렛은 반대로 **선택지를 8개로 좁히고 결정의 책임을 룰렛에 위임** 함으로써 문제를 해결합니다.

---

## 3. 가치 제안 (Value Proposition)

> **오늘 어디서 먹을지 정하기 어려운 순간, 맛집 룰렛은 주변 또는 선택 지역의 식당 후보를 자동으로 모아 게임처럼 한 곳을 골라줍니다. 검색보다 가볍고, 추천보다 재미있는 결정 도구입니다.**

## 4. 대상 사용자

| 페르소나 | 상황 | 맛집 룰렛이 해결하는 것 |
| --- | --- | --- |
| **점심 시간 직장인** | 매일 같은 동네에서 메뉴를 정해야 함. 동료끼리 "아무거나" 반복 | 결정 책임 위임 + 새로운 식당 발견 |
| **여행자 / 출장자** | 처음 가본 지역에서 식당을 골라야 함. 검색해도 뭘 골라야 할지 모름 | 지역 선택만으로 검증된 후보 자동 제시 |
| **모임 결정자** | 친구·연인·가족 모임에서 메뉴 결정 부담을 짊어짐 | 룰렛 결과를 핑계로 부담 없이 결정 |
| **컨텐츠 소비자** | 결정 자체를 게이미피케이션으로 즐기고 싶은 사용자 | 룰렛 회전 + 결과 공개 연출 |

## 5. 핵심 사용 시나리오

### 시나리오 A — 점심 직전, 사무실에서

1. 점심 시간 5분 전, 동료가 "오늘 뭐 먹지?" 라고 묻습니다.
2. 사용자는 휴대폰에서 맛집 룰렛을 엽니다.
3. **"내 주변"** 탭에서 검색 반경(1km / 2km / 5km 등)을 고릅니다.
4. 위치 사용을 허용하면 잠시 후 8개의 식당이 룰렛 위에 자리잡습니다.
5. 룰렛을 손으로 휘둘러 돌리거나 "돌리기" 버튼을 누릅니다.
6. 회전이 끝나면 당첨 식당의 사진·주소·전화번호가 카드로 펼쳐집니다.
7. "카카오맵에서 보기" 를 누르면 지도가 새 탭으로 열립니다.
8. 마음에 안 들면 "한 번 더!" 를 눌러 즉시 재시도합니다.

### 시나리오 B — 출장지에서, 저녁 약속

1. 처음 방문한 도시에서 저녁 식사 장소가 필요합니다.
2. **"지역 선택"** 탭에서 시/도와 시/군/구를 고릅니다.
3. 해당 지역의 식당 100여 곳 중 8곳이 룰렛에 올라옵니다.
4. 룰렛을 돌려 후보를 정하고 지도 링크로 위치를 확인합니다.

### 시나리오 C — 룰렛이 마음에 들지 않을 때

- 룰렛 8칸 자체를 바꾸고 싶다면 **"다른 8곳으로"** 를 눌러 같은 지역 내 다른 후보로 교체합니다.
- 현재 8칸에서 다시 돌리고 싶다면 **"한 번 더!"** 를 누릅니다.

## 6. 주요 기능

### 6.1 위치 입력 (탭 전환)

두 가지 입력 모드를 사용자가 직접 선택합니다.

- **내 주변** — 휴대폰·브라우저의 현재 위치를 기준으로 검색 반경(0.5km / 1km / 2km / 5km)을 선택합니다.
- **지역 선택** — 전국 17개 시/도와 산하 시/군/구를 드롭다운으로 선택합니다.

두 모드는 언제든 자유롭게 전환할 수 있습니다.

### 6.2 후보 자동 추출

사용자가 위치를 정하면 맛집 룰렛은 신뢰할 수 있는 **공공 관광 데이터(한국관광공사)** 에서 식당 정보를 가져옵니다. 최대 100곳의 후보 중에서 8곳을 무작위로 골라 룰렛에 올립니다.

- 광고나 협찬이 섞이지 않은 공공 데이터를 사용합니다.
- 매번 다른 8곳이 노출되므로 같은 지역이라도 룰렛을 채울 때마다 새로운 경험이 됩니다.

### 6.3 원형 룰렛

서비스의 핵심 인터랙션입니다.

- **드래그 발사**: 마우스나 손가락으로 룰렛을 잡고 휘두르면 휘두른 세기에 비례해 회전합니다.
- **버튼 발사**: 휘두르기가 어렵거나 한 손으로 조작할 때는 "돌리기" 버튼을 누르면 됩니다.
- **물리적인 감속**: 회전은 점차 느려져 자연스럽게 멈춥니다.
- **결과 정렬**: 회전이 멈출 때 화살표가 정확히 칸 한가운데를 가리키도록 미세 보정이 들어갑니다. **결과 카드에 표시되는 식당은 항상 화살표가 가리킨 칸의 식당과 동일합니다.**
- **칸을 지날 때마다 째깍 소리**가 나서 결과 직전의 긴장감을 줍니다.

### 6.4 결과 카드

회전이 멈춘 직후 결과 카드가 나타나며, 사용자에게는 아래 정보가 보입니다.

- 당첨 식당명과 사진 (사진이 없는 식당은 플레이스홀더로 대체)
- 주소
- 전화번호 (탭하면 바로 전화 걸기)
- 카테고리 코드
- **"카카오맵에서 보기"** 버튼 → 외부 지도가 새 탭에서 열림. 식당의 주소와 상호명을 함께 사용해 카카오맵 안에서 정확한 가게가 검색되도록 합니다.
- 결과 공개 시 컨페티 효과와 짧은 팡파레, 결과 카드 상단의 **"JACKPOT!"** 라벨이 가볍게 펄스합니다.

### 6.5 재시도

- **한 번 더!** — 현재 후보 8곳을 그대로 두고 룰렛을 한 번 더 돌립니다.
- **다른 8곳으로** — 같은 지역에서 새 후보 8곳을 뽑아 룰렛을 새로 채웁니다.

### 6.6 시각 컨셉 — 슬롯머신 콘솔

전체 화면은 **카지노 슬롯머신 콘솔을 모티프**로 한 단일 다크 톤으로 제공됩니다. "검색 도구가 아닌 게임 머신" 으로 인식되도록 의도된 디자인입니다.

- **머신 외형**: 위치 선택, 룰렛, 결과 카드 모두 황금색 베젤(테두리) 안에 놓이고, 베젤 위·아래에는 노란 LED 점이 순차적으로 깜빡입니다.
- **상단 라벨**: 각 화면 영역 상단에 `ROULETTE`, `INSERT COIN`, `WINNER` 같은 작은 LED 표시 라벨이 붙어 머신의 상태를 알려줍니다.
- **PUSH 버튼**: 후보 뽑기·룰렛 발사·재시도 같은 핵심 액션 버튼은 큰 빨간 돔 버튼으로, 누를 때 살짝 가라앉는 인터랙션이 있습니다.
- **반경 / 지역 선택**: 반경 칩과 시·도/시·군·구 선택기는 머신 내부 LED 패널 톤으로 표현되어, 선택된 항목은 노란빛으로 발광합니다.
- **로고**: 상단의 "MATJIP ROULETTE" 로고는 네온 글로우 톤으로 표시됩니다.

이번 버전은 다크 톤 단일 디자인으로 제공되며, 라이트 모드는 별도로 제공하지 않습니다. 머신 컨셉을 흐트리지 않고 시각적 일관성을 우선했기 때문입니다.

### 6.7 사운드 토글

- 회전음·당첨 팡파레를 끄거나 켤 수 있습니다.
- 설정은 다음 방문 시에도 유지됩니다.

### 6.8 반응형

- 휴대폰(좁은 세로 화면)·태블릿·데스크톱 모두 동일한 경험을 제공합니다.

| **친근한 한국어** | 모든 안내 문구는 격식 있되 친근한 정중체로 작성합니다. |

| **머신 일관성** | 상단 입력 영역부터 룰렛, 결과 카드, 빈 상태까지 동일한 슬롯머신 베젤·LED·돔 버튼 톤을 유지합니다. 한 화면 안에서 컨셉이 깨지지 않도록 디자인 합니다. |

## 8. 결과 카드에 노출되는 정보 (사용자 관점)

룰렛 결과로 사용자에게 보이는 정보의 범위를 정합니다. 이 목록 외의 정보는 노출하지 않습니다.

- 식당명
- 대표 이미지 1장 (없는 경우 플레이스홀더 아이콘)
- 도로명 주소
- 전화번호 (탭 시 전화 걸기)
- 카테고리 코드
- 외부 지도 링크 (카카오맵)

리뷰 점수, 가격대, 영업 시간, 메뉴 가격은 **이번 버전에서는 노출하지 않습니다**. (외부 지도 링크를 통해 확인하도록 유도)

## 9. 사용성 유의사항 (Known Caveats)

> 이 섹션은 운영팀·고객 응대·마케팅 카피 작성 시 미리 알고 있어야 할 항목들입니다.

- 사용자가 위치 권한을 허용하지 않은 경우 "내 주변" 결과가 표시되지 않을 수 있으며, 이때 사용자는 **"지역 선택"** 탭으로 자동 안내되어 지역을 직접 고를 수 있어야 합니다.

- 검색 반경이 좁거나 외곽 지역인 경우 후보 식당이 8곳보다 적게 잡힐 수 있으며, 이때 사용자는 **반경을 넓히거나 인근의 더 큰 행정 구역을 선택해야 할 수 있습니다.** 8곳 미만일 때는 룰렛 대신 안내 카드가 표시됩니다.

- 후보 데이터는 공공 관광 데이터를 기반으로 하므로 **데이터 갱신 시점과 실제 영업 상태가 다를 수 있습니다.** 영업 종료 또는 이전한 식당이 결과로 나올 가능성이 있으며, 사용자는 방문 전 전화나 지도 링크로 영업 여부를 확인하도록 안내됩니다.

- 결과 카드의 사진은 공공 데이터가 제공하는 경우에만 노출되며, **사진이 없는 식당은 일러스트 플레이스홀더로 대체됩니다.** 이는 데이터 품질의 한계이며, 식당 자체의 평가와는 무관합니다.

- 휴대폰 환경에서는 운영체제 정책상 **사용자가 화면을 한 번 터치한 이후부터** 회전음·팡파레가 활성화됩니다. 첫 진입 시 무음으로 시작될 수 있다는 점을 사용자가 알 필요는 없으나, 운영팀은 인지하고 있어야 합니다.

- 인터넷 연결이 불안정하거나 공공 데이터 서버가 일시적으로 느릴 경우 후보 불러오기가 지연되거나 실패할 수 있으며, **이때 사용자에게는 재시도를 권유하는 메시지가 표시되어야 합니다.** 일정 시간이 지나면 자동으로 시간 초과 안내가 나타납니다.

- 카카오맵 외부 링크는 새 탭에서 열리므로 일부 브라우저(특히 모바일)에서 **팝업 차단 설정에 따라 열리지 않을 수 있습니다.** 사용자에게는 브라우저 설정 확인을 안내합니다.

## 10. 비기능 요구사항 (사용자 관점)

- **속도**: 위치 선택 후 후보가 나타날 때까지 평균 1~2초 이내를 목표로 합니다.
- **모바일 우선**: 점심 시간대 실제 사용 환경이 휴대폰이므로 좁은 화면에서도 룰렛 휠이 잘 보이고 휘두를 수 있어야 합니다.
- **접근성**: 키보드만으로도 모든 기능을 사용할 수 있습니다. 화면 낭독기 사용자에게도 룰렛 상태와 결과가 음성으로 전달됩니다.
- **로그인·개인정보 수집 없음**: 위치 정보는 브라우저 안에서만 일시적으로 사용되고 서버에 저장되지 않습니다.
- **다국어**: 이번 버전은 한국어 단일 언어입니다.

## 11. 성공 기준

운영 1차 시점의 정성·정량 지표 후보입니다. (실 트래픽 확보 전이므로 목표 수치는 가설입니다.)

- **완주율 (Activation)** — 첫 진입 사용자 중 룰렛을 한 번 이상 돌리는 비율. 목표 60% 이상.
- **재시도율 (Engagement)** — 룰렛을 돌린 사용자 중 "한 번 더" 또는 "다른 8곳으로" 를 누르는 비율. 목표 40% 이상. 이 지표가 높을수록 "결정 도구" 보다 "엔터테인먼트" 가치가 큰 것으로 해석합니다.
- **지도 이동률 (Outcome)** — 결과 카드를 본 사용자 중 카카오맵 링크를 누르는 비율. 목표 50% 이상. 실제 방문 의도의 대리 지표입니다.
- **체류 시간** — 평균 1분 30초 이상.
- **재방문율** — 7일 내 재방문 사용자 20% 이상.

## 12. 의도적으로 만들지 않은 것

**현재 버전에는 들어가지 않은 기능** 입니다.

- 즐겨찾기·북마크
- 사용자가 입력하는 리뷰·평점
- 친구 초대 / 단체 룰렛
- 가격대·영업 시간·메뉴 필터
- 지도 위 시각화 (현재는 외부 지도 링크로 대체. **다음 프로젝트에서 직접 다룰 예정**)
- 카카오·구글 로그인
- 결제·예약
- 광고·협찬

## 13. 향후 확장 후보 (참고)

다음 단계 학습 과제에서 우선 검토할 만한 기능들입니다. 본 PRD의 요구사항은 아닙니다.

1. **지도 직접 표시** — 결과 카드 위 또는 별도 화면에 미니 지도. (다음 프로젝트 주제)
2. **카테고리 필터** — 한식/중식/카페 등 사용자가 직접 카테고리를 좁힐 수 있게 함.
3. **결과 공유** — 결과 카드를 이미지로 저장하거나 링크 공유.
4. **다국어** — 영어 / 일본어 / 중국어 (특히 관광객 대상).
5. **단체 모드** — 여러 사람이 같이 후보를 보고 룰렛을 돌리는 동기화 화면.
